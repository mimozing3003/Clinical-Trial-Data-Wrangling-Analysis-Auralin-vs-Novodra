# 🧪 Clinical Trial Data Wrangling & Analysis — Auralin vs Novodra

## 🚀 Project Overview
This project performs **end-to-end data wrangling and exploratory analysis** on a simulated Phase II clinical trial dataset for a new oral insulin drug **Auralin**, compared against **Novodra**.

The dataset mimics real-world healthcare data challenges such as:
- Missing values
- Inconsistent formats
- Redundant and messy tables
- Data integrity issues

The goal is to transform raw, unreliable data into a **clean, structured dataset** suitable for analysis and decision-making.

---

## 🎯 Objectives
- Assess data quality and tidiness issues
- Clean and standardize multiple datasets
- Merge patient, treatment, and adverse reaction data
- Analyze treatment effectiveness and safety
- Demonstrate real-world data engineering skills in healthcare

---

## 📂 Dataset Description

| File | Description |
|------|------------|
| `patients.csv` | Patient demographics and baseline data |
| `treatments.csv` | Treatment assignments and dosage |
| `adverse_reactions.csv` | Side effects and medical responses |

> ⚠️ Note: This dataset is simulated but designed to reflect real clinical trial complexity.

## 🔍 Data Wrangling Steps

### 1. Data Assessment
- Checked structure, data types, and missing values  
- Identified inconsistencies across multiple tables  
- Detected duplicate and invalid entries  

### 2. Data Cleaning
- Handled missing values  
- Fixed incorrect data types  
- Standardized column formats  
- Removed duplicates and anomalies  

### 3. Data Tidiness
- Ensured each variable is a column  
- Ensured each observation is a row  
- Restructured messy datasets into a clean format  

### 4. Data Integration
- Joined datasets:
  - Patients  
  - Treatments  
  - Adverse Reactions  
- Created a consolidated analytical dataset
